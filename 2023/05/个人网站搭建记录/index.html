个人网站搭建记录

Joey

2023-05-25 00:00:00

由于小小的纸质名片不能承载那么多的信息，而且纸质名片上的信息不能够实时的更新，我一直想着搞一个网页把自己的信息放上面，把自己的经历也放在上面，然后在纸质名片上放上一个网址就行。

·我的需求

（1）简洁、快速：缩小网站的资源大小是很有必要的，不仅能节约访问者的流量，而且还可以提高访问网页的速度。
（2）方便维护：之前自己找了个模板，每次修改图片和增删文章需要修改网页里很多地方，十分不方便，所以每次维护更新十分头疼。
（3）成本低：能白嫖就白嫖。

·方案选择

简单百度一下，个人网站的方案还是挺多的。结合我的需求来看，我最终选择了hexo + Github Page的方案，也是网上很多人使用的方案。hexo也提供了很多模板，节省了搭建的时间，对于前端小白十分的友好。我比较偏爱小而美的简约的设计风格，所以挑选了现在看到的网页模板。

·模板裁剪和修改

1、模板引入
我挑选的模板：minima，在它的github仓库里面有教程，而且网上对于hexo的教程也很多，按着教程，把一系列的文件弄到自己的电脑里。。
2、修改和个性化
由于,它是全个英文的模板，汉化一下看起来比较顺眼。汉化之前大致了解一下html语法和hexo的结构，方便对一些文件进行动手。
主要修改的地方有
~/hexo-blog/_config.yml
~/hexo-blog/themes/minima/_config.yml
~/hexo-blog/themes/minima/layout/..下EJS文件
~/hexo-blog/themes/minima/source/css/..下css文件
3、加评论功能
评论部分没有过多的要求，越简洁越好，所有人都可以发表留言，可以选择性的登录。对比了一些博客评论方案，其中valine和waline比较符合我的需求，不过网上对valine的隐私保护问题提出了质疑，不过，waline的后台使用国内版的leancloud的话需要一个审核过的域名，如果没有自己的域名，建议放弃，因为使用国际版的leancloud的话，不开启魔法上网，没法看到别人的留言。所以，现阶段，不准备买域名的我准备先使用valine，等后面买了域名再进行更新升级。
